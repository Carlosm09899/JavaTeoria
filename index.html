Ejemplo 1.8 pag 27

Promedio de Calificaciones de Alumno (Java)

Dada la matrícula y 5 calificaciones de un alumno obtenidas a lo largo del semestre, construya un programa en Java que lea los datos desde la consola e imprima la matrícula del alumno y el promedio de sus calificaciones.

Diagrama de flujo

Datos del Problema

  • Datos de Entrada: MAT, CAL1, CAL2, CAL3, CAL4, CAL5
  • Descripción de Datos (Tipos en Java):
    • MAT: int (representa la matrícula del alumno).
    • CAL1, CAL2, CAL3, CAL4, CAL5: double (representan las 5 calificaciones del alumno).
  • Salida Esperada:
    • Imprimir la matrícula (MAT).
    • Imprimir el promedio (PROMEDIO) de las 5 calificaciones.

Requisitos de Implementación (Java)

  • Crear una clase principal (ej. PromedioAlumno).
  • Usar la clase java.util.Scanner para leer los datos desde la consola.
  • Solicitar al usuario la matrícula (MAT).
  • Solicitar al usuario las 5 calificaciones (CAL1 a CAL5).
  • Calcular el promedio. La operación debe ser de punto flotante.
  • Imprimir la matrícula (MAT) y el promedio (PROMEDIO) en la consola usando System.out.println().

Ejemplo de uso esperado

(Al ejecutar el programa en la consola)

Ingrese la matrícula: 12345
Ingrese calificación 1: 10.0
Ingrese calificación 2: 9.5
Ingrese calificación 3: 8.0
Ingrese calificación 4: 7.5
Ingrese calificación 5: 9.0
--- Resultados ---
Matrícula: 12345
Promedio: 8.8

Notas rápidas (Java)

Recuerda que:

  • La clase Scanner es fundamental para leer datos del teclado.
  • Usa el tipo double para las calificaciones y el promedio para poder manejar decimales.
  • El tipo int es suficiente para la matrícula.
Tip: ¡No olvides cerrar el Scanner! Al final de tu programa, es una buena práctica cerrar el objeto Scanner usando scanner.close(). Esto libera los recursos del sistema que estaba utilizando para leer la entrada. Aunque en programas pequeños puede no parecer importante, es un hábito fundamental para evitar "fugas de recursos" (resource leaks) en aplicaciones más grandes.
Ejemplo 2.1 pag 55

Verificar Aprobación (Java)

Construya un programa en Java que, dado como dato la calificación de un alumno en un examen, escriba "Aprobado" en caso de que esa calificación sea mayor a 8.

Diagrama de flujo

Datos del Problema

  • Dato de Entrada: CAL
  • Descripción de Datos (Tipos en Java):
    • CAL: double (representa la calificación del alumno).
  • Salida Esperada:
    • Imprimir la cadena "Aprobado" solo si la calificación es estrictamente mayor que 8.
    • Si la calificación es 8 o menor, no se debe imprimir nada.

Requisitos de Implementación (Java)

  • Crear una clase principal (ej. VerificarAprobacion).
  • Usar la clase java.util.Scanner para leer los datos desde la consola.
  • Solicitar al usuario la calificación (CAL).
  • Utilizar una estructura condicional if para evaluar si CAL > 8.
  • Imprimir "Aprobado" usando System.out.println() solo si la condición es verdadera.

Ejemplo de uso esperado

(Ejecución 1: Calificación > 8)
Ingrese la calificación: 9.5
Aprobado

(Ejecución 2: Calificación <= 8)
Ingrese la calificación: 7.0
(No se imprime nada)

Notas rápidas (Java)

Recuerda que:

  • Este problema es un ejemplo de "Estructura Selectiva Simple".
  • Solo se realiza una acción si la condición es verdadera.
  • No se necesita una cláusula else, ya que no hay acción para el caso "falso".
Tip: La condición es "estrictamente mayor que" (>), no "mayor o igual que" (>=). Un alumno con 8.0 exacto no cumple la condición.

    • Imprimir la matrícula (MAT).
    • Imprimir el promedio (PROMEDIO).
    • Imprimir el estado: "Aprobado" o "No Aprobado".

  • Usar java.util.Scanner para leer la matrícula (int) y las 5 calificaciones (double).
  • Calcular el promedio (double).
  • Utilizar una estructura condicional if-else para evaluar si PROMEDIO >= 6.
  • Imprimir la matrícula y el promedio.

  • Datos de Entrada: 270 números enteros (NUM1, NUM2, ..., NUM270).
  • Descripción de Datos (Tipos en Java):
    • NUMi: int (cada número ingresado).
  • Salida Esperada:
    • La suma total de los números impares (long recomendado para evitar desbordamiento).
    • El promedio de los números pares (double).

Requisitos de Implementación (Java)

  • Usar java.util.Scanner para leer los números.
  • Usar un bucle for que se repita 270 veces.
  • Declarar e inicializar variables acumuladoras:
    • long sumaImpares = 0;
    • long sumaPares = 0;
    • int conteoPares = 0;
  • Dentro del bucle, leer un número (int num = scanner.nextInt();).
  • Usar el operador módulo (%) para verificar si num % 2 == 0 (par) o no (impar).
  • Acumular los valores según corresponda.
  • Después del bucle, calcular el promedio de los pares (double promedioPares).

Ingrese el número 2: 3
...

Notas rápidas (Java)

Recuerda que:

  • Este problema requiere una estructura repetitiva (bucle for).
  • El operador módulo (%) es clave para determinar si un número es par (num % 2 == 0).
  • Usa long para las sumas para evitar un posible desbordamiento (integer overflow) si los números son muy grandes.
Tip: ¡Cuidado con la división por cero! Antes de calcular sumaPares / conteoPares, siempre verifica que conteoPares sea mayor que 0. Si no lo es, el programa se detendrá con un error (ArithmeticException).

Contar Números Primos (Java)

Se dice que un entero positivo N es un número primo si los únicos enteros positivos que lo dividen son exactamente 1 y N. Diseñe un programa en Java que lea un número M, y obtenga y cuente todos los números primos menores a M.

Diagrama de flujo

Datos del Problema

  • Dato de Entrada: M
  • Descripción de Datos (Tipos en Java):
    • M: int (representa el número límite que se ingresa).
  • Salida Esperada:
    • Imprimir en la consola cada número primo que sea menor que M.
    • Imprimir el conteo total de los números primos encontrados.

Requisitos de Implementación (Java)

  • Usar java.util.Scanner para leer el entero M.
  • Usar un bucle for (externo) para iterar N desde 2 hasta M-1.
  • Para cada N, usar un bucle for (interno) para verificar si es primo.
  • Usar una variable boolean esPrimo = true; como bandera para cada N.
  • El bucle interno debe probar divisores desde 2 hasta la raíz cuadrada de N (Math.sqrt(N)).
  • Si se encuentra un divisor (N % divisor == 0), cambiar esPrimo a false y romper el bucle interno.
  • Después del bucle interno, si esPrimo sigue siendo true, imprimir N e incrementar un contador conteoPrimos.
  • Al final del programa, imprimir el conteoPrimos total.

Ejemplo de uso esperado

Ingrese el número límite (M): 20
Números primos menores que 20:
2
3
5
7
11
13
17
19
--- Resultados ---
Total de números primos encontrados: 8

Notas rápidas (Java)

Recuerda que:

  • Este problema requiere bucles anidados.
  • El 1 no se considera un número primo. Tu bucle externo debe empezar en 2.
  • El 2 es el primer número primo y el único par.
Tip de Optimización: Para verificar si un número N es primo, solo necesitas probar divisores hasta su raíz cuadrada (Math.sqrt(N)). Si no encuentras un divisor en ese rango, el número es primo. Esto hace tu programa mucho más rápido.

Almacenar los Primeros 30 Números Primos (Java)

Construya un programa en Java para almacenar en un arreglo unidimensional los primeros 30 números primos. Al final imprima el arreglo correspondiente.

Diagrama de flujo

Datos del Problema

  • Datos de Entrada: Ninguno. El número 30 es una constante.
  • Salida Esperada:
    • La impresión en consola del arreglo que contiene los primeros 30 números primos.

Requisitos de Implementación (Java)

  • Declarar un arreglo de enteros: int[] primos = new int[30];.
  • Declarar un contador para los primos encontrados: int conteoPrimos = 0;.
  • Declarar una variable para el número que se está probando: int num = 2;.
  • Usar un bucle while (conteoPrimos < 30) para buscar hasta llenar el arreglo.
  • Dentro del while, implementar la lógica para verificar si num es primo (usando un bucle for interno y una bandera boolean).
  • Si num es primo, guardarlo en el arreglo (primos[conteoPrimos] = num;) e incrementar conteoPrimos.
  • Incrementar num en cada iteración del while para probar el siguiente número.
  • Al finalizar el bucle while, imprimir el arreglo.

Recuerda que:

  • Este problema utiliza un arreglo (int[]).
  • El bucle externo debe ser while, ya que no sabes de antemano qué número será el primo número 30.
  • El bucle interno (para chequear primalidad) puede ser un for.
Tip: Para imprimir el contenido de un arreglo de forma legible, importa la clase java.util.Arrays y usa el método Arrays.toString(tuArreglo). Es mucho más fácil que imprimirlo elemento por elemento con un bucle.
Problema 4.17 pag 307

Matriz Transpuesta (Java)

Construya un programa en Java que, dada una matriz A(M x N) de tipo entero, calcule la transpuesta de dicha matriz. La transpuesta se obtiene al escribir las filas de la matriz A como columnas.

Diagrama de flujo

Datos del Problema

  • Datos de Entrada: M (filas), N (columnas), y luego M × N enteros.
  • Descripción de Datos (Tipos en Java):
    • int M (Filas, 1 ≤ M ≤ 50).
    • int N (Columnas, 1 ≤ N ≤ 30).
    • int[][] A = new int[M][N]; (Matriz Original).
  • Salida Esperada:
    • Una nueva matriz int[][] T = new int[N][M]; (Matriz Transpuesta).

Requisitos de Implementación (Java)

  • Usar java.util.Scanner para leer M y N.
  • Declarar la matriz original: int[][] A = new int[M][N];.
  • Usar bucles for anidados para leer los M × N elementos de A (fila por fila).
  • Declarar la matriz transpuesta: int[][] T = new int[N][M];.
  • Usar bucles for anidados (i para filas de A, j para columnas de A).
  • Asignar el valor transpuesto: T[j][i] = A[i][j];.
  • Usar bucles for anidados para imprimir la matriz transpuesta T.

Ejemplo de uso esperado

Ingrese el número de filas (M): 3
Ingrese el número de columnas (N): 4

Ingrese los elementos de la matriz A:
Ingrese fila 1 (4 elementos): -5 6 8 4
Ingrese fila 2 (4 elementos): 2 5 7 9
Ingrese fila 3 (4 elementos): 1 3 4 6

--- Matriz Transpuesta (4x3) ---
-5 2 1 
6 5 3 
8 7 4 
4 9 6

Notas rápidas (Java)

Recuerda que:

  • Este problema requiere arreglos bidimensionales (matrices).
  • La dimensión de la matriz transpuesta es la inversa de la original. Si A es de M x N, entonces T debe ser de N x M.
  • La clave del algoritmo está en intercambiar los índices: el elemento en la fila i y columna j de la matriz original va a la fila j y columna i de la transpuesta.
Tip: Para imprimir la matriz con formato, usa System.out.print(elemento + " ") dentro del bucle interno para los elementos de una fila, y un System.out.println() vacío después de que termine el bucle interno para pasar a la siguiente línea.

    4. Incrementar k: k += 2; (pasa de 1 a 3, 3 a 5, etc.).

Construya un programa en Java para almacenar en un arreglo unidimensional los 100 primeros números Fibonacci. Imprima al final el arreglo correspondiente.

Diagrama de flujo

Datos del Problema

  • Datos de Entrada: Ninguno. El número 100 es una constante.
  • Salida Esperada:
    • La impresión en consola del arreglo que contiene los primeros 100 números Fibonacci.

Requisitos de Implementación (Java)

  • Importar java.math.BigInteger y java.util.Arrays.
  • Declarar un arreglo de 100: BigInteger[] fibonacci = new BigInteger[100];.
  • Asignar los casos base:
    • fibonacci[0] = BigInteger.ZERO;
    • fibonacci[1] = BigInteger.ONE;
  • Usar un bucle for que itere desde i = 2 hasta 99.
  • Dentro del bucle, calcular el término actual: fibonacci[i] = fibonacci[i-1].add(fibonacci[i-2]);.
  • Al finalizar el bucle, imprimir el arreglo usando Arrays.toString(fibonacci).

Ejemplo de uso esperado

(Al ejecutar el programa, no se pide entrada)

Los primeros 100 números Fibonacci son:

Notas rápidas (Java)

Recuerda que:

  • El 100º número Fibonacci es gigantesco. Los tipos int y long no son suficientes y causarán un desbordamiento (overflow).
  • Es obligatorio usar la clase java.math.BigInteger para manejar números tan grandes.
  • Para operar con BigInteger, no puedes usar operadores como +. Debes usar sus métodos: BigInteger.ZERO, BigInteger.ONE y .add().
Tip: Para imprimir el arreglo de forma legible, importa java.util.Arrays y usa System.out.println(Arrays.toString(fibonacci));.

Datos del Problema

Gestión de Productos en Almacén (Java)

Una compañía distribuye N productos (1 ≤ N ≤ 150) a distintos comercios. Se almacena en un arreglo toda la información relacionada a su mercancía. Construya un programa en Java que pueda llevar a cabo las siguientes operaciones: venta, reabastecimiento, actualización de precio e informe sobre un producto.

Diagrama de flujo

Estructura de Datos

  • Se recomienda definir una clase Producto con los siguientes atributos:
    • int clave (CLA)
    • String descripcion (DES)
    • int existencia (EXI)
    • int minimo (MIN) - Mínimo a mantener en existencia.
    • double precioUnitario (PU)
  • El programa principal manejará un arreglo de objetos: Producto[] productos = new Producto[N];
  • Alternativamente, se podrían usar arreglos paralelos (uno para cada campo), aunque es menos práctico.

Operaciones Requeridas

  1. Venta de un producto:
    • Datos: clave del producto, cantidad vendida.
    • Buscar el producto por su clave en el arreglo.
    • Si se encuentra y hay existencia suficiente:
      • Actualizar la existencia (restar la cantidad vendida).
      • Verificar si la nueva existencia está por debajo del mínimo. Si es así, mostrar una advertencia.
    • Si no se encuentra o no hay existencia suficiente, informar del error.
  2. Reabastecimiento de un producto:
    • Datos: clave del producto, cantidad comprada.
    • Buscar el producto por su clave.
    • Si se encuentra, actualizar la existencia (sumar la cantidad comprada).
    • Si no se encuentra, informar del error.
  3. Actualizar el precio de un producto:
    • Datos: clave del producto, porcentaje de aumento (ej. 10 para 10%).
    • Buscar el producto por su clave.
    • Si se encuentra, calcular el nuevo precio (precioActual * (1 + porcentaje / 100.0)) y actualizar el campo precioUnitario.
    • Si no se encuentra, informar del error.
  4. Informar sobre un producto:
    • Dato: clave del producto.
    • Buscar el producto por su clave.
    • Si se encuentra, imprimir todos sus datos (clave, descripción, existencia, mínimo, precio).
    • Si no se encuentra, informar del error.

  • La implementación central es la búsqueda del producto por clave en el arreglo.
